أريد تنفيذ مرحلة Production Presentation لمنصة ZENO قبل النشر النهائي.

الهدف:
تحويل الواجهة من مظهر نموذج هاكاثون أو نظام تجريبي إلى منتج تقني مالي احترافي ومتناسق، مع الحفاظ على الدقة وعدم الادعاء بوجود اعتماد رسمي أو اتصال مصرفي حقيقي.

لا تغيّر منطق النظام الأساسي ولا تحذف البيانات أو المزايا التي تعمل.

========================================
1. إزالة العبارات التجريبية المتكررة
========================================

ابحث في جميع ملفات المشروع عن العبارات التالية أو ما يشابهها:

- نموذج تجريبي
- بيانات تجريبية
- Mock AI
- Mock AI Mode
- Prototype Mode
- Demo
- Demo Account
- حساب تجريبي
- محاكاة فقط
- لأغراض العرض فقط
- لا توجد حركة مالية حقيقية
- نظام هاكاثون
- Coming Soon
- ZENO Secure Demo

لا تعرض هذه العبارات بشكل متكرر داخل الصفحات الرئيسية أو البطاقات أو الشريط الجانبي.

استبدلها بصياغات احترافية مناسبة حسب السياق، مثل:

- محرك زينو الذكي
- بيئة زينو
- مركز الموافقات
- تمت معالجة الطلب
- تم تأكيد الإجراء
- حساب العميل
- حساب الإدارة
- حساب المحلل
- نظام التوصيات
- البيانات المالية
- منصة زينو
- محرك تجربة العميل
- حالة النظام: متصل
- حالة الخدمة: تعمل بكفاءة

مهم:
لا تستخدم عبارة توحي أن النظام معتمد رسميًا أو مرتبط حاليًا ببنك حقيقي.

========================================
2. تنبيه قانوني واحد بدل التكرار
========================================

لا تكرر التنبيهات في كل بطاقة وصفحة.

أنشئ صفحة أو نافذة باسم:

"حول المنصة والسياسات"

وتحتوي نصًا مهنيًا واضحًا:

"زينو منصة تقنية مالية لتطوير تجربة العميل المصرفية. تعتمد النسخة الحالية على بيئة تشغيل وبيانات محاكية، ولا تنفذ معاملات مالية عبر حسابات مصرفية فعلية. صُممت البنية لتكون قابلة للتكامل مستقبلًا مع البيئات المصرفية والواجهات المعتمدة بعد استكمال المتطلبات الأمنية والتنظيمية."

أضف رابطًا صغيرًا لهذه الصفحة في:

- Footer
- صفحة تسجيل الدخول
- قائمة الإعدادات

لا تجعل التنبيه يطغى على تجربة المستخدم.

========================================
3. عدم الادعاء بالاعتماد الرسمي
========================================

احذف أو عدّل أي نص يدعي:

- اعتماد البنك المركزي السعودي.
- شراكة رسمية مع البنك المركزي السعودي.
- موافقة مصرفية رسمية.
- تكامل فعلي مع بنك.
- تنفيذ تحويلات أو مدفوعات حقيقية.
- أن النظام جاهز للإطلاق المصرفي الإنتاجي دون مراجعة.

استخدم بدلًا منها:

"مصمم للمواءمة مع الأطر التنظيمية السعودية."

أو:

"قابل للتكامل مع البيئات المصرفية بعد استيفاء المتطلبات التنظيمية والأمنية."

لا تستخدم شعار البنك المركزي السعودي أو أي شعار جهة رسمية دون تصريح.

========================================
4. تحديث مسميات النظام
========================================

استبدل المسميات الحالية كما يلي:

"Mock AI Mode"
إلى:
"محرك زينو الذكي"

"Prototype Mode"
إلى:
"ZENO Experience Engine"

"حساب عميل تجريبي"
إلى:
"حساب العميل"

"إدارة زينو التجريبية"
إلى:
"إدارة زينو"

"تمت تجريبيًا"
إلى:
"تم التأكيد"

"تأكيد تجريبي"
إلى:
"تأكيد الإجراء"

"بيانات العرض"
إلى:
"بيانات المنصة"

"إعادة بيانات العرض"
إلى:
"استعادة الإعدادات الأساسية"

"حذف بيانات العرض المحلية"
إلى:
"حذف البيانات المحلية"

"بيانات تجريبية"
إلى:
"بيانات النظام"

"النظام نموذج تجريبي"
إلى:
احذفها من الواجهة الرئيسية، وضع التوضيح الكامل فقط في صفحة حول المنصة والسياسات.

========================================
5. تحديث تجربة تسجيل الدخول
========================================

اجعل صفحة تسجيل الدخول تبدو كنظام مالي احترافي.

العنوان:

ZENO — The Silent Bank
زينو — البنك الصامت

النص:

"تجربة مصرفية استباقية تفهم احتياجات العميل، وتقدم التوصية المناسبة في الوقت المناسب، مع بقاء القرار النهائي بيده."

استخدم المسميات:

- دخول العميل
- دخول الإدارة
- دخول المحلل

لا تستخدم:

- حساب العرض
- تسجيل دخول تجريبي
- Demo Account

يمكن إبقاء أزرار الدخول السريع لأغراض اللجنة، لكن اجعل عنوانها:

"الدخول حسب الدور"

وفي أسفل الصفحة ضع رابطًا صغيرًا:

"حول المنصة والسياسات"

========================================
6. تحديث مساعد زينو
========================================

استبدل عبارة:

"المساعد الحالي يعمل بردود تجريبية"

بصياغة احترافية:

"مساعد زينو يحلل سياق العميل ويقدم شروحات وتوصيات مرتبطة ببياناته المالية."

غيّر الشارة:

"Mock AI Mode"

إلى:

"ZENO AI"

أو:

"محرك زينو الذكي"

لا تدّع أن المساعد مرتبط بنموذج خارجي أو نظام مصرفي إن لم يكن ذلك مطبقًا.

احتفظ بالحماية من:

- XSS
- HTML injection
- إدخال الرسائل الفارغة

========================================
7. تحديث مركز الموافقات
========================================

استخدم حالات احترافية:

- بانتظار الموافقة
- تم التأكيد
- مرفوض
- ملغى
- قيد المراجعة

عند تأكيد الإجراء اعرض:

"تم تأكيد الإجراء بنجاح."

لا تعرض عبارة:

"تمت العملية تجريبيًا."

لكن لا تخصم رصيدًا فعليًا ولا تدّع إرسال تحويل مصرفي حقيقي.

إذا كانت العملية محاكاة داخلية فقط، اجعل منطقها:

- تحديث حالة الطلب.
- تسجيل الحدث في سجل النشاط.
- عرض إشعار نجاح.
- عدم الاتصال بأي بوابة دفع أو بنك.

========================================
8. تحديث لوحة الإدارة والمحلل
========================================

احذف كلمة "تجريبي" من عناوين البطاقات والجداول.

استخدم:

- مؤشرات تجربة العميل
- أداء التوصيات
- معدل القبول
- متوسط الثقة
- الوقت الموفر
- رحلة العميل
- تحليلات الاستخدام
- نقاط الاحتكاك
- قواعد محرك التوصية

أضف ملاحظة صغيرة داخل صفحة "حول المنصة والسياسات" بأن المؤشرات الحالية مبنية على بيئة محاكاة، بدل تكرار ذلك في كل لوحة.

========================================
9. تحسين الهوية الرسمية
========================================

تأكد من استخدام شعار ZENO الرسمي في:

- صفحة تسجيل الدخول.
- Sidebar.
- Topbar.
- Favicon.
- شاشة التحميل.
- صفحة 404.
- صفحة الخطأ.

وتأكد من تطبيق خطوط ثمانية الثلاثة على كامل المشروع:

thmanyahserifdisplay-Bold:
- العناوين الرئيسية.
- عناوين الصفحات والبطاقات.
- اسم زينو.

thmanyahseriftext-Medium:
- الفقرات.
- الشروحات.
- التوصيات.
- رسائل المساعد.

thmanyahsans-Regular:
- الأزرار.
- القوائم.
- الجداول.
- الحقول.
- الأرقام.
- الشارات.
- عناصر النظام.

لا تستخدم أي خطوط أخرى إلا fallback.

========================================

========================================

"فشل تحميل بيانات العرض."

اكتب:
"تعذر تحميل البيانات. حاول مرة أخرى."

بدل:
"النظام التجريبي غير متاح."

اكتب:
"الخدمة غير متاحة مؤقتًا."

بدل:
"Coming Soon."

اكتب:
"قيد التطوير."

ولا تترك أي زر بلا وظيفة؛ اجعله يعمل أو disabled مع توضيح "قيد التطوير".

========================================
11. سجل النشاط
========================================

أضف سجل نشاط احترافي يعرض:

- تم تسجيل الدخول.
- تم فتح التوصية.
- تم قبول التوصية.
- تم إرسالها إلى مركز الموافقات.
- تم تأكيد الإجراء.
- تم تحديث إعداد الخصوصية.
- تم تعديل قاعدة توصية.

لا تسجل كلمات المرور أو البيانات الحساسة.

========================================
12. الفحص النهائي
========================================

ابحث داخل المشروع كاملًا عن الكلمات التالية:

demo
mock
prototype
experimental
تجريبي
محاكاة
أغراض العرض
هاكاثون
Admin1234
localhost

راجع كل نتيجة يدويًا.

المسموح أن تظهر هذه المفاهيم فقط في:
- صفحة حول المنصة والسياسات.
- README التقني.
- ملفات التطوير الداخلية عند الضرورة.

يجب ألا تظهر في الواجهة الرئيسية أو رحلة العميل إلا عند الحاجة القانونية أو التقنية.

نفذ:

npm install
npm run lint
npm run typecheck
npm run build
npm start

اختبر:

- تسجيل دخول العميل.
- الإدارة.
- المحلل.
- التوصيات.
- الموافقات.
- مساعد زينو.
- الخصوصية.
- الجوال.
- تحديث الصفحة.
- الشعار.
- الخطوط.
- حالات الخطأ.

في النهاية أعطني:

1. جميع النصوص التي تم تغييرها.
2. جميع الملفات التي تم تعديلها.
3. العبارات التي بقيت في صفحة السياسات فقط.
4. نتيجة Build.
5. تأكيد عدم وجود ادعاء اعتماد رسمي أو اتصال مصرفي حقيقي.
6. تأكيد أن الواجهة أصبحت بهوية إنتاجية احترافية.
